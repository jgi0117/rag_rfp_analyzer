# notebooks

이 디렉터리는 RFP 데이터와 전처리 결과를 탐색하기 위한 분석 노트북과 요약 문서를 보관합니다. 운영 파이프라인의 핵심 코드는 `src/`와 `experiments/`에 있으며, 이곳의 자료는 데이터 품질 확인과 실험 방향 결정에 참고하는 용도입니다.

## 주요 파일

`EDA.ipynb`는 원본 RFP 메타데이터와 파일 구성을 점검한 탐색 분석 노트북입니다. 파일 매칭 여부, 결측치, 사업금액, 일정, 텍스트 길이, 빈출어, PDF 구조 후보 등을 확인합니다.

`EDA.md`는 `EDA.ipynb`의 주요 결과를 Markdown으로 정리한 문서입니다. 데이터 규모, 품질 이슈, HWP to PDF 변환 결과, PDF 표/이미지 후보, 다음 RAG 단계 제안이 포함되어 있습니다.

## 사용 목적

노트북은 새 전처리 방식이나 평가 전략을 적용하기 전에 데이터의 상태를 빠르게 확인하는 데 사용합니다. 특히 파일명 매칭, 문서 길이, 결측 메타데이터, 반복 서식 문구처럼 검색 품질에 영향을 줄 수 있는 요소를 사전에 확인하는 역할을 합니다.

## 주의사항

- 노트북 실행 결과로 생기는 대용량 데이터, 이미지, 임시 산출물은 Git에 올리지 않습니다.
- 정식 파이프라인에 반영할 로직은 노트북에만 두지 말고 `src/` 또는 `experiments/`로 옮겨 재사용 가능하게 관리하세요.
- 데이터 경로는 루트 기준 상대 경로를 사용하는 것이 좋습니다.

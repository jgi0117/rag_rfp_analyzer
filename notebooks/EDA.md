# RFP 데이터 EDA 요약

작성 기준: `notebooks/EDA.ipynb`

## 1. 분석 목적

`data/raw`에 저장된 RFP 메타데이터와 원본 문서 파일을 점검해 이후 RAG 파이프라인 구축 전에 확인해야 할 품질 이슈를 파악했다. 주요 확인 범위는 다음과 같다.

- 메타데이터와 실제 파일의 매칭 여부
- 결측치, 중복, 빈 본문 텍스트 여부
- 발주기관, 파일 형식, 사업금액, 일정 분포
- 제목/요약/본문 텍스트 길이와 빈출어
- 업무 유형 키워드 그룹 분포
- PDF 변환 결과와 PDF 내 표/이미지/도형 후보

## 2. 입력 데이터

| 항목 | 값 |
|---|---:|
| 메타데이터 행 수 | 100 |
| 메타데이터 컬럼 수 | 12 |
| 원본 파일 수 | 101 |

추가 원본 파일은 `고려대학교_차세대 포털·학사 정보시스템 구축사업.docx`이며, 같은 사업의 PDF 파일이 메타데이터와 매칭되어 있다.

## 12. 다음 RAG 단계 제안

- 메타데이터와 파일 경로 매칭 결과를 인덱싱 전 고정한다.
- 공고번호/공고차수가 없는 행은 파일명 또는 별도 ID로 출처 식별자를 만든다.
- HWP 변환 실패 3건은 한컴에서 수동 열기/다른 저장 옵션으로 재시도한다.
- 본문 길이가 지나치게 짧은 문서는 원문 추출 품질을 별도 검토한다.
- 반복적인 RFP 서식 문구는 청킹 전에 제거하거나 낮은 가중치로 처리한다.
- 업무 유형 키워드 그룹은 검색 필터와 평가용 라벨로 활용한다.
- PDF 표/그래프 후보는 샘플 검수 후 표 추출 파이프라인을 별도로 설계한다.

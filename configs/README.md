# configs

이 디렉터리는 RAG 실험에 필요한 공통 설정과 문서 목록을 관리합니다. 실험 스크립트는 기본적으로 `configs/base.yaml`을 먼저 읽고, 개별 실험 YAML에서 필요한 항목만 덮어쓰는 방식으로 동작합니다.

## 주요 파일

- `base.yaml`: 프로젝트 이름, 원천 데이터 경로, 이미지 저장 여부, 평가 대상 문서 목록을 정의하는 공통 설정 파일입니다.
- `experiments/`: 임베딩 모델, 생성 모델, 청킹 전략, 벡터 DB 저장 위치, 평가 결과 저장 경로처럼 실험마다 바뀌는 값을 관리합니다.

## 설정 구조

`project`는 프로젝트 이름과 기본 대상 문서명을 담습니다. 단일 문서 실험을 할 때 기본값으로 사용되며, 여러 문서를 평가할 때는 `documents` 목록이 우선됩니다.

`path`는 데이터 입출력 경로를 정의합니다. `file_dir`은 PDF 파일들이 들어 있는 디렉터리이고, `image_dir`은 PDF에서 이미지 추출을 켰을 때 이미지가 저장되는 위치입니다. 현재 `data/`와 `outputs/`는 `.gitignore`에 포함되어 있으므로 원본 데이터와 실행 산출물은 Git에 올라가지 않습니다.

`documents`는 평가에 사용할 RFP 목록입니다. 각 항목의 `document_id`는 검색 필터, Ground Truth 매칭, 평가 결과 CSV에서 문서를 구분하는 핵심 키로 사용됩니다. `file_name`은 `path.file_dir` 아래의 실제 PDF 파일명과 일치해야 합니다.

## 수정 시 주의사항

- 문서를 추가할 때는 `document_id`가 중복되지 않도록 관리하세요.
- `file_name`은 실제 파일명과 완전히 같아야 합니다.
- Git에 올릴 수 없는 로컬 경로나 개인 PC 절대 경로는 설정 파일에 넣지 않는 편이 좋습니다.
- 실험 결과 경로는 개별 실험 YAML의 `output` 항목에서 관리하는 것을 권장합니다.
